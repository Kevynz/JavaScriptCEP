Hoisting

Hoisting em JavaScript é o comportamento em que declarações de variáveis e funções são "elevadas" ao topo de seu escopo (global ou de função) durante a compilação, antes da execução do código. Principais pontos: Variáveis com var: São içadas, mas apenas a declaração. O valor inicial não é içado, resultando em undefined se acessadas antes da inicialização. Funções declaradas: Tanto a declaração quanto o corpo da função são içados, permitindo que a função seja chamada antes de sua definição no código. Variáveis com let e const: São içadas, mas não podem ser acessadas antes de serem declaradas devido à "temporal dead zone" (zona morta temporal). Hoisting afeta como o código se comporta em tempo de execução, especialmente ao tentar acessar variáveis ou funções antes de sua posição no código.

Arrow Functions

Arrow Functions em JavaScript são uma maneira mais concisa de declarar funções, introduzidas no ES6. Elas apresentam algumas diferenças importantes em relação às funções normais (funções declaradas com function), especialmente em termos de comportamento do this e sintaxe. Diferenças principais: Sintaxe mais curta: Arrow functions permitem escrever funções com menos código, tornando-as mais compactas, especialmente para funções simples de uma linha. this léxico: Ao contrário das funções normais, as arrow functions não criam seu próprio contexto de this. Elas herdam o valor de this do escopo onde foram definidas (escopo léxico). Isso é útil em callbacks e métodos de objetos, evitando a necessidade de bind() ou salvar this em uma variável. Sem arguments: Arrow functions não têm o objeto arguments, que nas funções normais contém todos os argumentos passados. Se você precisar de todos os argumentos, precisará usar o rest parameter (...args) com arrow functions. Não podem ser usadas como construtores: Arrow functions não podem ser usadas com new para criar objetos, pois elas não têm o comportamento de função construtora. As funções normais, por outro lado, podem ser usadas como construtoras. Sem prototype: Arrow functions não têm o atributo prototype, diferentemente das funções normais. Isso significa que elas não podem ser usadas para criar métodos a serem compartilhados entre instâncias de objetos. Resumo: Sintaxe: Arrow functions têm uma sintaxe mais curta. this: Arrow functions herdam o valor de this do escopo em que foram criadas, enquanto funções normais criam seu próprio contexto de this. Objeto arguments: Só está disponível em funções normais. Construtor: Arrow functions não podem ser usadas como construtoras. Prototype: Arrow functions não possuem o atributo prototype.

Desestruturação (Destructuring)

Desestruturação (ou destructuring) em JavaScript é uma sintaxe que permite extrair valores de arrays ou propriedades de objetos de forma mais concisa e intuitiva. Essa técnica melhora a legibilidade do código e facilita a manipulação de dados complexos. Principais características: Desestruturação de Arrays: Permite extrair elementos de um array e atribuí-los a variáveis de maneira direta. Por exemplo, você pode atribuir os primeiros elementos de um array a variáveis individuais. Desestruturação de Objetos: Facilita a extração de propriedades de um objeto, permitindo que você crie variáveis com o mesmo nome das propriedades do objeto ou especifique nomes diferentes. Exemplos de uso: Com arrays, você pode facilmente extrair os primeiros elementos sem a necessidade de acessar índices. Com objetos, você pode acessar diretamente as propriedades que deseja, tornando o código mais limpo e claro. Resumo: A desestruturação em JavaScript é uma técnica poderosa que simplifica a extração de dados de arrays e objetos, melhora a legibilidade do código e torna a manipulação de dados mais eficiente.

Clojure

Clojure é uma linguagem de programação funcional que roda na Java Virtual Machine (JVM). Ela é projetada para ser simples, eficiente e altamente expressiva, permitindo a criação de software de maneira concisa e flexível. Principais características: Imutabilidade: Clojure enfatiza o uso de estruturas de dados imutáveis, o que significa que uma vez criadas, as estruturas não podem ser alteradas. Isso ajuda a evitar efeitos colaterais indesejados e torna o código mais previsível e fácil de entender. Programação Funcional: Clojure é uma linguagem funcional, o que significa que ela trata funções como cidadãos de primeira classe. Isso permite que funções sejam passadas como argumentos, retornadas de outras funções e armazenadas em estruturas de dados.

Resumo: Clojure é uma linguagem funcional e imutável que roda na JVM, projetada para simplificar a programação e permitir a criação de software de maneira concisa e expressiva, aproveitando as capacidades do Java e promovendo um estilo de programação robusto e eficaz.
